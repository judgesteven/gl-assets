Quick Setup

  1. Download the component files: mission.js, mission.css, and assets.css
  2. Include the CSS files in your HTML head section
  3. Include the JavaScript file before the closing body tag
  4. Update the configuration in mission.js with your GameLayer API credentials (see below)
  5. Add mission card HTML with the structure shown above
  6. Initialize the component - it auto-initializes when the page loads

API Configuration

Required Configuration

Update these values in mission.js:

  • Base URL: https://api.gamelayer.co/api/v0 (or your custom endpoint)
  • API Key: Your GameLayer API key
  • Default Player: Your player ID
  • Account ID: Your GameLayer account ID

Configuration Example

const GAMELAYER_CONFIG = {
    baseURL: 'https://api.gamelayer.co/api/v0',
    apiKey: 'your-actual-api-key-here',
    defaultPlayer: 'your-player-id',
    accountId: 'your-account-id',
    debugMode: false
};

GameLayer API Integration

API Endpoints

  • Get Mission: GET /missions/{id}?account={accountId}
  • Get All Missions: GET /missions?account={accountId}

Required Headers

{
    "Content-Type": "application/json",
    "Accept": "application/json",
    "api-key": "your-api-key"
}

Data Structure

The component expects mission data with the following structure:

  • imgUrl - Mission image URL (optional)
  • active.to - Mission end time for countdown
  • reward.points - Points awarded upon completion
  • reward.credits - Credits awarded upon completion
  • objectives.events[0].currentcount - Current progress
  • objectives.events[0].count - Total required

Component Features

  • Responsive Design: Mobile-first approach with iOS-inspired styling
  • Real-time Data: GameLayer API integration for live mission data
  • Multiple States: Active, completed, upcoming, locked, and expired missions
  • Event-driven: Custom events for easy integration with your application
  • Accessibility: ARIA attributes and keyboard navigation support
  • Cross-platform: Works on iOS, Android, and web platforms
  • Self-contained: No external dependencies required
